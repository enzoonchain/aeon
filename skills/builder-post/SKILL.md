# Builder Post Skill

Generate X/Twitter posts that build the project narrative in 0xNairolf style. Builder perspective — not storyteller, not marketer. Ship updates, share data, ask questions, build in public.

## Core Principles

1. **Observation first** — lead with data or fact, not opinion
2. **Builder voice** — "we shipped" not "we're excited to announce"
3. **Data over hype** — numbers > adjectives
4. **No filler** — no "here's what you missed", no "thread 🧵", no "big news 🚀"
5. **Signature close** — "this is base rn" / "🟦"
6. **Max 1-2 emoji per post** — never spam
7. **Short sentences** — max 2-3 lines per block

## Post Types

### 1. Milestone Post
**When:** shipping a feature, hitting a number, completing work
**Template:**
```
[what we shipped]

[data point]

[why it matters]

this is base rn 🟦
```

**Example:**
```
delegation v1 is live.

first 3 agents connected. 50 DIEM delegated. revenue split executing on-chain.

unused compute → productive work → yield for delegators.

this is base rn 🟦
```

**When:** sharing on-chain data, usage stats, market numbers
**Template:**

## What NOT to do

- ❌ "here's what you missed today 🧵"
- ❌ "we're excited to announce"
- ❌ "big news 🚀"
- ❌ "thread (1/7)"
- ❌ "stay tuned"
- ❌ "DYOR" / "not financial advice"
- ❌ emoji spam (✅🟢🔴)
- ❌ tables with pipe syntax
- ❌ corporate speak

## What TO do

- ✅ lead with data or observation
- ✅ short punchy sentences
- ✅ conversational: "bro", "ngl", "yea well"
- ✅ builder voice: "we shipped", "we found", "we're building"
- ✅ signature close: "this is base rn" / "🟦"
- ✅ max 186 chars for lead tweets
- ✅ threads for deeper dives (max 5 tweets)
