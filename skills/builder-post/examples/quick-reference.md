# Builder Post — Quick Reference

## Voice Checklist

Before posting, check:
- [ ] Lead with data or observation (not opinion)
- [ ] Builder voice ("we shipped" not "we're excited")
- [ ] At least one number
- [ ] No filler ("here's what you missed", "big news", "🚀")
- [ ] Max 1-2 emoji
- [ ] Signature close ("this is base rn" / "🟦")
- [ ] Under 186 chars

## Post Type Selector

| Situation | Post Type |
|-----------|-----------|
| Shipped a feature | Milestone |
| Sharing stats/data | Data Drop |
| Research findings | Alpha |
| Work in progress | Build |
| Asking community | Engagement |

## Key Phrases (use these)

- "we just shipped"
- "the numbers:"
- "here's what we found"
- "we're building"
- "question for builders"
- "that's the play"
- "this unlocks"
- "on-chain proof"

## Key Phrases (never use)

- "here's what you missed"
- "we're excited to announce"
- "big news 🚀"
- "thread 🧵 (1/7)"
- "stay tuned"
- "DYOR"
- "not financial advice"
- "to the moon"

## Thread Structure (if needed)

1. Hook tweet (data/observation)
2. Context (what it means)
3. Implication (why it matters)
4. Our take (builder perspective)
5. CTA or question (optional)

Max 5 tweets per thread.
